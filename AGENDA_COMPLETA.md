## üîÑ Confronto Tecnologie (Approfondimenti)

### React vs JavaScript Puro
- **Virtual DOM**: React ottimizza il rendering automaticamente
- **Component-driven**: Organizzazione modulare vs script procedurali
- **Riutilizzo**: Componenti vs funzioni globali
- **Ecosystem**: npm packages infiniti vs librerie vanilla limitate
- **Learning curve**: Steeper ma pi√π produttivo a lungo termine

---

### React Hooks vs Class Components (Storico)
- **useState** vs `this.state`: Pi√π semplice e diretto
- **useEffect** vs `componentDidMount/Update`: Un solo hook per tutto
- **Custom Hooks**: Riutilizzo logica impossibile con classi
- **Performance**: Meno overhead, bundle pi√π piccoli

---

### Context API vs Redux
| Aspetto | Context API | Redux Toolkit |
|---------|-------------|---------------|
| **Complessit√†** | Semplice, built-in | Pi√π strutturato, libreria esterna |
| **DevTools** | Basilari | Redux DevTools avanzati |
| **Performance** | Re-render potenziali | Ottimizzato con selectors |
| **Learning curve** | Immediata | Ripida ma potente |
| **Use case** | Theme, auth, dati semplici | State complesso, async logic |
| **Boilerplate** | Minimo | Ridotto con Toolkit |

---

### TypeScript vs JavaScript in React
- **Type safety**: Catch errori in sviluppo, non in produzione
- **Autocomplete**: IDE suggestions migliorate
- **Refactoring**: Sicuro e veloce
- **Documentation**: Tipi = documentazione automatica
- **Team**: Codice pi√π manutenibile in team
- **Trade-off**: Setup iniziale + sintassi extra
